# GeoHex v3 Specification

AI向けドメインナレッジ。GeoHex v3 の仕様をまとめる。

## 概要

GeoHex v3 は地球表面を正六角形のグリッドで分割する地理空間インデックスシステム。
六角形は正方形や長方形と比べ、中心からの距離が均等で隣接セル数が6（矩形は4〜8）という特性を持ち、空間的な近傍検索に有利。

- 公式サイト: https://sites.google.com/site/geohexdocs
- 座標系: WGS 84 (EPSG:4326)
- 投影法: Webメルカトル（内部計算用）

## レベルとセルサイズ

| Level | セルサイズ (m) | 用途例       |
| ----- | -------------- | ------------ |
| 0     | ~5,000 km      | 大陸レベル   |
| 3     | ~180 km        | 国・地域     |
| 5     | ~20 km         | 都市圏       |
| 7     | ~2.2 km        | 市区町村     |
| 9     | ~250 m         | 町丁目       |
| 11    | ~28 m          | 建物         |
| 13    | ~3 m           | ピンポイント |
| 15    | ~0.3 m         | 最高精度     |

セルサイズの計算式: `H_BASE / 3^(level + 3)` where `H_BASE = 20037508.34`（メルカトルの半周長）

## コード体系

GeoHex コードは `[文字2つ][数字列]` の形式。

- **先頭2文字**: `A-Z a-z`（52文字）の中から選択。2文字の組み合わせでグリッド座標の上位桁をエンコード。
  - `c1 = floor(n / 30)`, `c2 = n % 30` で `H_KEY[c1] + H_KEY[c2]` に変換
  - `H_KEY = "ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz"`
- **残りの数字**: `0-8`（9進数）。各桁がグリッド座標のX/Y成分を2つのbase-3数字としてエンコード。
- **コード長**: `level + 2`。Level 0 なら2文字、Level 7 なら9文字。

### コード例

- `XM` — Level 0（2文字）
- `XM488` — Level 3（5文字）
- `XM4885413` — Level 7（9文字、東京駅付近）

### バリデーション

- 最小長: 2文字
- 先頭2文字: 大小英字（`A-Za-z`）
- 3文字目以降: `0-8` の数字のみ

## エンコード

1. 緯度/経度をWebメルカトル座標に変換
2. メルカトル座標を六角グリッド座標 `(gx, gy)` に変換
   - `unitX = 6 * size`, `unitY = 6 * size * tan(30°)`
   - `posX = (mercX + mercY / tan(30°)) / unitX`
   - `posY = (mercY - tan(30°) * mercX) / unitY`
3. 六角形の境界判定で最近接セルを決定
4. 日付変更線付近のワールドラップ補正
5. グリッド座標を balanced ternary → base-9 → コード文字列に変換

## デコード

1. コード文字列を先頭2文字と数字部分に分離
2. 先頭2文字から数値を復元
3. base-9 → base-3ペア → X/Y成分のデインターリーブ
4. balanced ternary（0=−1, 1=0, 2=+1）でグリッド座標を復元
5. グリッド座標 → メルカトル座標 → 緯度/経度

## 六角形の形状

フラットトップ（水平辺が上下）の正六角形。6頂点で構成される。

```
     ┌───┐
    /     \
   /       \
   \       /
    \     /
     └───┘
```

頂点順序: 左 → 左上 → 右上 → 右 → 右下 → 左下（時計回り）

## 特記事項

- **ワールドラップ**: 日付変更線（±180°）を跨ぐケースでは座標補正が必要。グリッド座標の `x - y` の差が `3^(level+2)` に達すると折り返し処理が発生する。
- **浮動小数点精度**: encode → decode の往復で微小な誤差が生じうる（±0.0001° 程度）。
- **プレフィックス階層性なし**: GeoHash と異なり、コードの部分一致が空間的な親子関係を表すわけではない。
